Ignorando arquivos - Arquivo .gitignore

  • Ao commitar arquivos em um repositório pode acontecer de haver alguns arquivos que não queremos ou devemos incluir ao commit;
  • Para evitar que esses arquivos sejam commitados por engano, podemos usar um arquivo especial que indica ao git que determinados arquivos devem ser ignorados, e não rastreados;
  • Para tal usamos o arquivo .gitignore;

Arquivo .gitignore

  • Um arquivo .gitignore é um arquivo de texto plano no qual cada linha especifica um padrão de arquivos ou diretórios a ignorar;
  • Geralmente, o colocamos no diretório raiz de um repositório, mas é possivel colocá-lo em outro lugar(mas não é recomendado);

Quais arquivos devem ser ignorados?

  • É recomendado ignorar arquivos de tipos como:
    • Arquivos de log;
    • Arquivos que contenham senhas, credenciais ou informações confidenciais;
    • Arquivos de sistema e temporários;
    • Arquivos gerados, como executáveis de teste;
    • Saídas compiladas(executáveis. etc.);
    • Arquivos pessoais;
    • Entre outros;

Arquivo .gitignore global

  • Podemos criar um arquivo global para ignorar arquivos e diretórios em todos os projetos simultaneamente;

touch ~/.gitignore

Acima estamos criando o arquivo .gitignore dentro da pasta do projeto ou diretório.

git config --global. core.excludesFile

Agora todos os repositórios git ignoram os arquivos colocados no arquivo ~/.gitignore que é global;

Padrões de correspondência

  • Podemos colocar o nome dos arquivos e diretórios que serão excluídos no .gitignore seguindo alguns padrões;
  • Nome de arquivos literais:
    • .tempArchive;
  • Diretórios:
    • temp/
  • Caracteres - curinga:
    • *0 Ou mais caracteres(exceto /);
    • ? Um único caractere(exceto /);
    • ! Negação(negar ignore);
    • # Comentários no arquivo gitignore;

Exemplo:

Exemplo de comando Função
archTest Ignora o arquivo ou diretório archText
*.log Ignora todos os arquivos com extensão .log
Arch Ignora todos os arquivos iniciados com Arch
?rq Ignora todos os arquivos não importando o primeiro caracteres, mas que possua rq na sequência
dirTest/ Ignora o diretório dirTest e seus arquivos
**/dirTest Ignora todos os diretórios de nome dirTest
!exit.log Não ignora o arquivo exit.log

.gitignore - Dicas importantes

  • Não é necessário comitar o próprio arquivo .gitignore, a não ser que queira compartilhar suas entradas com outros usuários no projeto, mas é recomendável compartilhar este arquivo com outros usuários do projeto;
  • Evite criar muitos arquivos .gitignore. Se possivel, trabalhe com arquivo global;

touch .gitignore

No exemplo acima estamos criando o arquivo .gitignore no projeto.

echo archive1 >> .gitignore

Agora informamaos o arquivo a ser ignorado. Com isso o arquivo archive1, não estará mais na stage area, pois foi ignorado.
