Comando git push

  • O comando git push é usado para enviar o conteúdo do repositório local para um repositório remoto;
  • Assim os arquivos do projeto armazenados no computador, ficam hospedados remotamente;
  • O comando push transfere commits do repositório local a um repositório remoto;
  • Resumindo, os arquivos ficam hospedados no GitHub, ou em outro repositório remoto;

git push git@github.com:kesley-dev/project.git master

Precisamos informar o endereço SSH do repositório, e a branch para qual o projeto será enviado.

Efetuando isso, os arquivos do repositório local e seus commits serão enviados para o repositório remoto do GitHub.

Sendo assim, sempre que efetuarmos alterações e commits no nosso repositório local, devemos realizar o comando git push para enviar as atualizações ao repositório do GitHub, e lembrando que temos que inserir o endereço SSH do repositório e a branch para qual será enviada as atualizações.

Simplificando o push - git push

  • Ao efetuarmos o comando git push devemos informar o endereço SSH do repositório;
  • Porém, podemos simplificar a operação utilizando o comando git remote;

git remote add origin git@github.com:kesley-dev/project.git

Com isso a origin(origem) será configurada para o repositório indicado e caso quisermos visualizar com mais detalhes, usamos o comando git remov -v. Por exemplo:

git remote -v

Agora que temos a origin(origem) configurada, podemos realizar o git push de forma simples ao repositório do GitHub.

git push origin master

Colocando autor no commit

  • Ao efetuar um commit, podemos colocar o autor que o está realizando;
  • Vale ressaltar que as informações do autor serão mostradas no GitHub;

git commit -m "Texto do H2 alterado" --author="nomedousuário ‹emaildousuário@gmail.com›"

Após isso, enviamos o projeto para o repositório remoto do GitHub com o git push.

git push origin master
