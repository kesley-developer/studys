Fundindo branches - git merge

  • Ao efetuarmos alterações nas branches do projeto, devemos fundir essas atualizações na branch principal, que é denominada com master ou main;
  • Chamamos esse processo de merge;

git merge

  • O comando git merge, fundi alterações feitas em um branch a outro;

Criando branch

git branch menu

Podemos efetuar commits e alterações na branch menu, e ao finalizar todo o processo devemos fundir a branch menu na branch padrão chamada master ou main.

Fundindo branch

git merge menu

Com o exemplo anterior, as edições feitas na branch menu serão fundidas a branch master.

Resolvendo conflitos

  • Ao trabalharmos em equipe durante o desenvolvimento de um projeto, poderá haver conflitos de implementações;
  • Ou seja, cada desenvolvedor do projeto poderá fazer um merge(fundir) de recursos similares um do outro, o que acaba gerando um conflito de código;
  • Com isso, o git não sabe qual commit inserir durante a realização de uma fusão com git merge;
  • O git não tem um comando específico para resolução de conflitos, por isso devemos corrigir manualmente o problema e comunicando a equpe, ou utilizamos simplesmente git commit e analisamos as informações;

git commit
