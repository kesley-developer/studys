Pull Request(PR)

  • Pull Request(PR), quer dizer solicitação de puxar em sua tradução;
  • É uma forma de propor alguma alteração no código de um repositório;
  • Sendo assim, os desenvolvedores que tenham permissões de escrita nele, podem revisar a alteração proposta e discutir em cima dela, antes de aprovar a ter aquela alteração de fato incorporada a branch principal(master ou main);
  • Dessa forma podemos analisar quais branches serão fundidas ao repositório remoto do GitHub;
  • Fazendo um Pull Request(PR), podemos facilitar a resolução de conflitos de código e implementações;

1 - Enviar alterações para o repositório do GitHub

git push git@github.com:kesley-dev/project.git my_branch

2 - Acessar Repositório do GitHub e analisar informações

Com isso informações serão mostradas no GitHub, e devemos clicar em Compare & Pull Request que seremos direcionados para uma sessão onde poderemos visualizar os commits e alterações feitas ao projeto. Caso não tenhamos conflitos de código, podemos realizar a merge(fundir) da branch secundária para a branch principal que é reconhecida como main ou master.

De forma resumida, um Pull Request(PR) é simplesmente uma solicitação para efetuar alterações e implementações ao projeto, por isso é imporante durante o trabalho em equipe, pois terá interação entre os membro da equipe, para saber o que deve ou não ser adicionado ao projeto.
