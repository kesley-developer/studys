GitHub

  • O GitHub é um serviço de host(hospedagem) para repositórios git;
  • Quando desenvolvemos um projeto, os seus arquivos ficam armazenados em um repositório local, que simplesmente é o computador;
  • Caso quisermos compartilhar o repositório do projeto para os demais desenvolvedores, colocamos este repositório em um servidor remoto;
  • O Github é um tipo de servidor remoto, logo podemos hospedar projetos e arquivos de nossa máquina local;
  • Outro exemplo de serviço de hospedagem de projetos: Bitbucket;

Acima podemos ver o funcionamento de forma simplificada do Git e GitHub, quando trabalhamos de forma combinada.

Assim que efetuamos um push ao repositório do GitHub, outros desenvolvedores também podem pegar e visualizar atualizações mais recentes em relação ao projeto. Dessa forma cada membro da equipe poderá contribuir com o desenvolvimento do projeto, tendo acesso ao repositório remoto e suas atualizações utilizando o pull. Vale ressaltar que para efetuar o processo, devemos ter uma conta e repositório no GitHub.

Passo a passo para criar e configurar repositório

  • 1 - Acessar o GitHub e a conta
  • 2 - Criar repositório e suas definições
  • 3 - Configurar SSH - Uma chave do computador para autorizar a troca de informações do PC e o GitHub
    • 3.1 - Criar a SSH no terminal do git(git bash) e inserir no GitHub
    • 3.2 - Clicar em SSH and GPG KEYS
    • 3.3 - Acessar configurações do usuário
    • 3.4 - Definir configuração do SSH no GitHub
  • 4 - Configurar endereço SSH do repositório do GitHub no Git
    • 4.1 - Copiar endereço SSH do repositório no GitHub
    • 4.2 - Colar endereço SSH no git e configurá-lo

Seguindo o passo a passo corretamente, estaremos prontos para trabalhar com o diretório do repositório remoto do GitHub.
