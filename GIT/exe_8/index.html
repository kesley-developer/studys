Trabalhando com repositórios - git init

  • O comando git init inicia o repositório git;
  • Podemos escolher o diretório do projeto para iniciar o repostório git;

Repositório

  • Repositório é um local de armazenamento onde o projeto e suas alterações são mantidos;
  • Na prática, é uma pasta comum no sistema para cada projeto gerenciado pelo git, um repositório é configurado;
  • Sendo assim, para cada projeto novo, seguimos os seguintes passos:
    1. Criar um diretório para armazenar o projeto;
    2. Entrar no diretório(opcional);
    3. Inicializar o repositório git(git init);

Diretórios

  • Após o passo a passo para gerenciamento do projeto no git, é criado um subdiretório oculto chamado .git que guardará os dados de alteração e snapshots;
  • A área fora do diretório .git é o diretório de trabalho, onde ficam armazenados os arquivos que iremos manipular;
  • Ou seja, é a área onde interagimos diretamente com os arquivos do projeto;

Staging area

  • É o local para onde os arquivos vão antes de um snapshot ser criado;
  • Somente os arquivos da staging area vão para um snapshot;
  • Assim selecionamos os arquivos desejados no diretório de trabalho;

Commits

  • Snapshot do projeto em um determinado momento, com informações sobre o autor do conteúdo em quem o realizou;
  • O estado anterior do projeto é denominado "pai", commits são ligados entre si por conexões pai-filho, e um conjunto de commits relacionados entre si é chamado de branch(ramo);
  • Um commit pode ter dois pais se for criado pela merge(mescla) de dois branches;
  • Identificamos um commit pelo seu nome, que é uma string de 40 caracteres obtida pelo hash(SHA1) do commit;
