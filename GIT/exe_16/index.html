Criando branches - git branch

  • Um branch no git é simplesmente um ponteiro móvel para um commit;
  • O nome do branch padrão no git é master;
  • Conforme começamos a fazer commits, recebemos um branch master que aponta para o último commit que foi feito;
  • Cada vez que fazemos um novo commit ele avança automaticamente;
  • Para criar uma nova branch, utilizamos git branch "nomedabranch"

Com o diagrama acima temos um exemplo de comno funciona uma branch, que é praticamente uma cópia do projeto original, e a partir destas cópias diversos desenvolvedores podem trabalhar e criar uma nova feature(recurso) sem alterar o projeto princiapl que é o main ou master;

git branch

  • Com o comando git branch, podemos criar uma nova branch;
  • Lembrando que podemos navegar entre branches e alterar o ponteiro móvel para uma branch, usando o comando git checkout "nomedabranch"

git branch test

Assim será criada a branch chamada test, que é o local onde podemos fazer alterações e exclusões, sem interferir no projeto principal que é o main ou master.

Para alterar a branch, utilizamos o comando git checkout. Por exemplo:

git checkout test

Com o exemplo acima, a branch foi alterada para a test, sendo assim qualquer commit ou edição não afetará a branch master.

Também podemos visualizar todas as branches criadas com o comando git branch. Por exemplo:

git branch

Deletando branches

  • Podemos deletar branches com o comando git branch -d "nomedabranch"
  • Lembrando que antes de deletar uma branch, devemos fazer uma merge;
  • Isso significa que devemos fundir as alterações feitas no branch dentro do projeto princiapl, que é o master ou main;
  • Para deletar uma branch, usamos git branch -d "nomedabranch";

git branch -d test

Caso quisermos deletar a branch de qualquer maneira e sem detralhes, utilizamos o -D maiúsculo.
