Comandos inicias do Git - Três estados

  • Após a instalação do Git, devemos realizar algumas configurações para o usuário;
  • Para isso, abrimos o Git bash e digitamos alguns comando para configurar;
  • Precisamos seguir os três estados para fazer o versionamento corretamente, que são os seguintes estados: modificado, preparado e consolidado;

Definindo nome do usuario

git config --global user.name "Kesley Keven"

Definindo email do usuário

git config --global user.email "meuemail@gmail.com"

Acima definimos o nome de usuário com o comando user.name e o email com user.email.

Agora devemos configurar o diretório raíz caso seja necessário.

Verificando diretório

pwd

Podemos criar, navegar ou excluir arquivos utilizando os comando de terminal, tais como: cd. dir, etc...

Iniciando git no diretório(Estado modificado)

git init

Verificando status

git status

Com o comando git status, verificamos informações dos arquivos, e conseguimos saber se o arquivo sofreu modificação ou está pronto para um commit.

Adicionando arquivo ao rastreamento - staged(Estado consolidado)

git add archive.txt

Após está etapa, o proximo passo será gravar as alterações feitas ao repositório ou seja, faremos um commit.

git commit -m "Aqui colocamos uma mensagem do por que está sendo feito o commit"

Com -m podemos inserir uma mensagem ao commit

Verificando log

git log

Com este comando podemos verificar informações de cada commit feito com autor, horário e mensagem.

Lembrando que qualquer alteração feita ao arquivo do repositório, devemos preparar o arquivo para commit.

Preparando arquivo para commit(Estado preparado)

git add *

Com isso movemos os arquivos para o estado preparado, portanto agora realizamos o passo a passo para ser feito o estado consolidado(commit).

git commit -m "Alterações foram feitas ao arquivo"
