Revertendo commits - git revert

  • Eventualmente teremos a necessidade de reverter um commit;
  • Um exemplo disso, são erros cometidos nos arquivos;
  • Desfazer um commit é basicamente comitar exatamente o oposto do commit em si;
  • O git, na prática, cria um novo commit com as mudanças opostas;
  • Para efetuar está operação, utilizamos o comando git revert;

git revert

  • Com o git revert, podemos desfazer as mudanças em um commit, mantendo alterações posteriores intactas;
  • Os arquivos retornam ao mesmo estado com se o commit revertido nunca tivesse existido;
  • O git revert é em si, um commit, por isso não é necessário fazer mais nada após executar esse comando;
  • Precisamos especificar o nome do commit que será revertido, para obter essa informação usamos git lo;
  • Para reverter o último commit, usamos o comando git revert HEAD;
  • Caso quisermos cancelar a operação de reversão, utilizamos git revert --abort;
  • Lembrando que temos algumas regras, que são:
    1. Deve-se trabalhar em um diretório de trabalho limpo, executando sempre git add e git commit antes de tentar reverter um commit;
    2. Só devemos reverter um commit caso sejá necessário ou tenha erros;

git log --oneline

Com git log --oneline podemos visualizar informações dos commits de forma simplificada.

Agora com as informações dos commits, basta especificarmos no comando git revert o nome do commit para fazer a reversão.

git revert a10fd65
