Fazendo fork de um repositório

  • No GitHub quando clicamos em fork no repositório de um outro usuário, ele pega o repositório completo dessa pessoa e copia para a nossa conta git;
  • No git podemos editar esses arquivos e depois devolver a essa pessoa com as nossas eidções, e se caso o usuário aceitar, as alterações também entram no repositório dele;
  • Com tudo, caso quisermos contribuir com novas ideias em projetos Open Source(Código Aberto), realizamos um fork;

1 - Clicar em fork no repositório que desejamos contribuir

Com isso será criada uma cópia do repositório onde foi feito o fork, ao nosso repositório.

2 - Clonar repositório para nossa máquina com o git - git clone

git clone git@github.com/kesley-dev/project.git

Efetuando isso, teremos o repositório em nossa máquina com todos os seus commits e informações.

3 - Enviar alterações para o nosso repositório - git push

git push origin master

Utilizando o git push, enviamos as alterações para o nosso repositório, porém a nossa objeção é contribuir com o projeto ao qual foi feito fork, e para isso devemos fazer um Pull Request(PR).

4 - Criar um Pull Request(PR)

Seguindo e prosseguindo com o Pull Request(PR), basta analisarmos corretamente as informações mostradas e criá-lo.

A partir disso, a solicitação será enviada para o criador do projeto ao qual foi feito o fork, e caso ele aceite o Pull Request(PR), as edições serão implementadas ao projeto. Dessa maneira deixaremos nossa contribuição.
