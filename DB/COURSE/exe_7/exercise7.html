Ajustando a estrutura do banco de dados;

  • Pudemos observas nos exemplos anteriores, como criamos uma tabela no banco de dados;
  • Porém, devemos fazer ajustes a tabela para evitarmos diversos tipos de erros, como a repetição de um mesmo registro ou valor deste registro;
  • Para iniciar o passo a passo devemos definir uma chave primária;
  • Chave primária, também conhecida como Primary Key ou simplesmente PK, é um campo ou conjunto de campos com valores exclusivos por toda a tabela;
  • Com o conceito de PK(primary key), os valores da chave podem ser usados para se refirir aos registros inteiros, porque cada registro tem um valor diferente para a chave;
  • Cada tabela só pode ter uma chave primária;

Exemplo de tabela:

PESSOAS
id nome nascimento sexo peso altura nacionalidade
1 "Dados do nome" "Dados do nascimento" "Dados do sexo" "Dados do peso" "Dados da altura" "Dados da nacionalidade"
2 "Dados do nome" "Dados do nascimento" "Dados do sexo" "Dados do peso" "Dados da altura" "Dados da nacionalidade"

Acima temos a base de dados chamada pessoas, juntamente com a tabela. Neste tabela temos o id, cujo local é a chave primária de um registro. Nesta chave primária não podemos ter repetição de valores, pois ela é praticamente a identificação da coluna.
