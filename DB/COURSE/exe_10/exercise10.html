Evitando repetição de dados(UNIQUE)

  • É muito comum ter registros que possuem alguns dados parecidos;
  • Mas dependendo do tipo de campo do registro, o valor deve ser único, como o CPF de uma pessoa;
  • Para isso, usamos unique durante a criação ou alteração de uma coluna em uma tabela;
field type null key default extra
id int(11) NO PRI null
nome varchar(30) NO null
cpf varchar(12) NO UNI null

Observando acima temos o cpf com a key UNI. Isso significa que o campo não poderá repetir valores em seus registros.

Lembrando que chave primária é diferente de um valor único. A chave primária é a identificação de um registro, enquanto unique evita a repetição de um mesmo valor em um campo.
