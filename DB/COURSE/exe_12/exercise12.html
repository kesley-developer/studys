Gerenciando cópias de segurança MYSQL

  • Durante a manipulação de um banco de dados, erros podem acontecer, como a exclusão de uma tabela ou registro;
  • Mas é fundamental criar uma cópia, ou fazer um backup para recuperar dados perdidos, pois dados apagados não retornam;
  • O backup também é conhecido com dump;
  • Para fazer o backup ou dump, precisamos exportar a base de dados desejada e os seus arquivos .sql;
  • Não so podemos exportar, mas também importar arquivos .sql para incluir ao banco de dados;
